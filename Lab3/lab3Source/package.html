This package contains the code base for the third design pattern lab.

Introduction

Search for // YOUR CODE HERE in the skeleton.

In this laboration you will make the graphics of a snake game flexible by using structural design patterns. The game will be extended with graphical modules that you will make usable. In the beginning of every section you must write a short text (2-5 sentences) explaining your choice of design pattern. Ask the lab assistant if you can not find a design pattern that fits. The graphical modules are constructed so that it will take some thinking to make them usable in the game.

Usage

The snake can either be controlled by the arrow keys on the keyboard or by an automatic demo controller. There are also several possible graphical modules to use for displaying the snake, background, and bonuses.

Purpose

The purpose is to learn how to code with structural patterns and choose from a pattern dictionary the pattern most suited in a given situation.

Readings

The chapters on structural design patterns in GoF p. 137-217.

Assignments

First, make the ControllerKey usable. Edit in Snake.java, setKeyControll.

Second, Renderer is a common interface of all Graphics modules. Graphics3D has totally different method names than the Renderer interface. Therefore it needs to be adapted (hint intended).

Third, ModuleA, ModuleB, and ModuleC have the functionality for displaying a background, snake, and bonus, but they have totally different interfaces than the Renderer. Moreover, each module does one thing, but you need one object that can do all three things. Hint: Façade?

Fourth, decorate (subtle hint!) a renderer such that you can display the shadow of the snake and of the bonus. The shadow applies to the current renderer, regardless if it is the original, the 3D, or the ABC.

Hand in

Hand in all classes (.java-files only) that you have created or modified.
